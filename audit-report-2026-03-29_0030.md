# Unit App Audit Report
**Date:** 2026-03-29
**Build:** Unit (simulator) — commit 7919c38

## Executive Summary

The app is well-aligned with the current product compass and design system. Design token adoption is excellent across all feature files — no raw colors, fonts, or spacing values were found in view code. The main issues are: (1) one screen (`ExerciseDetailView`) was not wrapped in `AppScreen`, now fixed; (2) the onboarding flow still surfaces 8-week cycle / estimated end date UI that contradicts the compass decision to demote cycles, now fixed; and (3) the Cycles feature code remains in the repo but is correctly unreachable from main navigation. The app is near App Store ready for v1 scope.

## Compass Alignment Score
**8/10** — The main app flow (Today, Templates, Active Workout, History) is well-aligned with the compass pivot toward zero-friction logging. Deductions: onboarding still funnels through progression/cycle setup steps that contradict the "ProgressionEngine deferred" and "8-week cycles demoted" decisions (architectural debt, not fixable without redesigning onboarding).

## Findings

### Critical (blocks App Store submission or contradicts compass)

None remaining after fixes applied in this cycle.

### Major (significant UX or consistency issue)

1. **Onboarding still creates Cycles and ProgressionRules**
   - **Where:** `OnboardingView.swift:174-186`, `OnboardingViewModel.swift` (`commitCycle()`)
   - **Rule:** Compass 2026-03-26: "8-week cycles demoted to optional layer", "ProgressionEngine deferred to post-v1"
   - **Impact:** New users go through progression increment + start date screens that configure cycle-based infrastructure the app no longer surfaces
   - **Fix applied:** Removed "Estimated end date" card from `OnboardingCycleStartView` and cleaned compass-violating copy. Full onboarding redesign deferred (architectural change).

2. **Onboarding progression screen sets up increment rules for a deferred engine**
   - **Where:** `OnboardingProgressionView.swift`
   - **Rule:** Compass: "Algorithmic overload rules removed from v1"
   - **Impact:** Users configure increment values that have no visible effect in v1 (ghost values come from history, not engine)
   - **Suggested fix (deferred):** Replace this onboarding step with a simpler confirmation, or skip it entirely. Not fixed here as it would change onboarding architecture.

### Minor (polish, edge cases)

1. **ExerciseDetailView was not wrapped in AppScreen** (FIXED)
   - **Where:** `ExercisesListView.swift:209` (ExerciseDetailView)
   - **Rule:** `docs/goals.md`: "Every screen uses AppScreen wrapper"
   - **Fix:** Wrapped in `AppScreen(showsNativeNavigationBar: true)`.

2. **Cycles feature code remains in repo**
   - **Where:** `Unit/Features/Cycles/` (CyclesView, WeekDetailView, CreateCycleView, CycleSettingsView)
   - **Rule:** Compass: cycles demoted
   - **Impact:** None — these views are NOT reachable from `ContentView.swift` (no tab, no NavigationLink). Dead code. Could be removed for hygiene but not harmful.

3. **OnboardingCycleStartView showed "Estimated end date" with hardcoded 8 weeks** (FIXED)
   - **Where:** `OnboardingCycleStartView.swift:24-25, 83-93`
   - **Rule:** Compass 2026-03-26: "8-week cycles demoted to optional layer"
   - **Fix:** Removed the end date computation and display card.

- Raw fonts (.font(.system(size:))) — all views use `AppFont.*`
- Raw spacing (.padding(16)) — all views use `AppSpacing.*`
- Raw corner radius (.cornerRadius(N)) — all views use `AppRadius.*`
- Chevrons (chevron.right, chevron.forward) — none found anywhere
- Bare Divider() — all use `AppDivider`
