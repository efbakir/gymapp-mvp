## Violations found and fixed

### 1. Missing `reduceMotion` guards on `withAnimation` (HIG / AGENTS.md)

AGENTS.md requires: _"guard animated transitions with `@Environment(\.accessibilityReduceMotion)`"_. Four `withAnimation` calls lacked this guard:

| File | Line | Animation |
|------|------|-----------|
| `ActiveWorkoutView.swift` | 689 | `.easeInOut(duration: 0.18)` — exercise index transition |
| `CalendarTabView.swift` | 89 | `.easeInOut(duration: 0.2)` — date selection toggle |
| `OnboardingExercisesView.swift` | 186 | `.spring(response: 0.22, ...)` — drag reorder (via `ExerciseReorderDropDelegate`) |
| `TemplateDetailView.swift` | 105 | `.easeInOut(duration: 0.18)` — edit mode toggle |

**Fix:** Added `@Environment(\.accessibilityReduceMotion) private var reduceMotion` to each view. Guarded each `withAnimation` with `reduceMotion ? nil : <animation>`. For `OnboardingExercisesView`, the animation lives in a separate `DropDelegate` struct — `reduceMotion` is passed through as a property.

## Architectural notes (not fixed — flagged only)

### 2. Native `TabView`/UITabBar on root screens

`ContentView.swift` uses SwiftUI's native `TabView` with `.tabItem { Label(...) }`, which renders the native UITabBar. The banned patterns list specifies: _"Native UITabBar visuals on root screens → `UnitTabBar` via app-shell `safeAreaInset`"_. The `UnitTabBar` and `UnitTabItem` components exist in `DesignSystem.swift` but are not wired into the root shell.

**Recommendation:** Migrate to `UnitTabBar` in a dedicated PR. This is an architectural change that needs careful testing (tab selection state, hiding during active workout, etc.).

### 3. Several screens not wrapped in `AppScreen`

`goals.md` states _"Every screen uses AppScreen wrapper"_. The following screens use direct `ScrollView` + `.appNavigationBarChrome()` or custom layouts instead of `AppScreen`:

- `CyclesView` — uses `NavigationStack` + custom toolbar
- `WeekDetailView` — uses `ScrollView` + `.appNavigationBarChrome()`
- `PaywallView` — uses custom `ZStack` + `ScrollView` (modal)
- `CycleSettingsView` — uses `NavigationStack` + `Form` (justified for Form-based views)

**Recommendation:** Migrate `CyclesView` and `WeekDetailView` to `AppScreen` in a future pass. `PaywallView` is a modal with custom layout; `CycleSettingsView` uses `Form` which doesn't compose inside `AppScreen`.
