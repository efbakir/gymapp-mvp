## Findings

### Critical (blocks App Store submission or contradicts compass)

None.

### Major (significant UX or consistency issue)

1. **AGENTS.md was stale** ‚Äî Described Unit as "Adaptive Periodization Engine", referenced `AtlasLog/` folder paths (actual: `Unit/`), listed `AtlasTheme` as token source (actual: `DesignSystem.swift`), and prescribed "Target vs. Actual" UI (superseded by ghost values).
   - **Rule:** `docs/product-compass.md` decision log 2026-03-26 ‚Äî primary promise is zero-friction logging, not adaptive coach.
   - **Fix:** Updated AGENTS.md: project description, folder structure, critical rules, and conventions now reflect the compass pivot and current codebase. Fixed this cycle.

### Minor (polish, edge cases)

1. **TemplateDetailView not wrapped in AppScreen** ‚Äî Uses `List` directly for drag-and-drop reorder support.
   - **File:** `Unit/Features/Templates/TemplateDetailView.swift`
   - **Rule:** `docs/atomic-design-system.md` ‚Äî "New full-screen flows should compose inside AppScreen"
   - **Note:** Structural constraint from List-based drag/drop. Flagged for awareness, not fixed.

2. **PaywallView not wrapped in AppScreen** ‚Äî Uses custom `ZStack`/`ScrollView` layout.
   - **File:** `Unit/Features/Subscription/PaywallView.swift`
   - **Note:** Intentional full-bleed design. Flagged for awareness.

3. **Legacy Cycles views unreachable but still in project** ‚Äî `CyclesView`, `WeekDetailView`, `CreateCycleView`, `CycleSettingsView` exist but are not navigable from the main tab bar.
   - **Files:** `Unit/Features/Cycles/*.swift`
   - **Rule:** Compass 2026-03-26 ‚Äî cycles demoted to optional layer.
   - **Note:** Dead code retained for data compatibility. Consider removing in a future cleanup.

4. **Onboarding still creates Cycle objects** ‚Äî `OnboardingViewModel.commit()` creates a `Cycle` and `ProgressionRule` records. New sessions from TodayView have `cycleId: nil`, so these records are unused in the template workflow.
   - **Files:** `Unit/Features/Onboarding/OnboardingViewModel.swift:288-314`
   - **Rule:** Compass ‚Äî ProgressionEngine deferred to post-v1.
   - **Note:** Data creation side effect; doesn't affect UX. Architecture change to remove, so deferred.

5. **ProgressionEngine references in ActiveWorkoutView** ‚Äî Queries `progressionRules` and `cycles`; computes targets when `session.cycleId` is non-nil and `weekNumber > 0`. Inert for template-based sessions.
   - **File:** `Unit/Features/Today/ActiveWorkoutView.swift:20-21, 49-51, 63-69, 740-747`
   - **Note:** Backward-compatible fallback for existing cycle data. Not user-facing in normal flow.
