### Minor (polish, edge cases)

1. **Dead code: `OnboardingCycleStartView`** — `Unit/Features/Onboarding/OnboardingCycleStartView.swift` is not referenced from any route in `OnboardingView`. It exists only as a standalone file with a `#Preview`. No functional impact but adds codebase noise.
   - **Rule:** Scope fences — cycle views should not be in active feature paths (`AGENTS.md`).
   - **Fix:** Delete file or mark as legacy explicitly.

2. **PaywallView does not use `AppScreen` wrapper** — `Unit/Features/Subscription/PaywallView.swift` uses a raw `ZStack + ScrollView` instead of `AppScreen`. Layout needs (no nav bar, full-bleed) justify this, but it should be flagged per `docs/goals.md` ("Every screen uses `AppScreen` wrapper").
   - **Fix:** Justified exception — no change needed, but document the exception.

3. **TemplateDetailView uses raw `List` instead of `AppScreen`** — `Unit/Features/Templates/TemplateDetailView.swift` uses `List` with custom styling and toolbar. The drag-to-reorder and List row management require this layout. All tokens are correctly used.
   - **Fix:** Justified exception — List-based layouts are incompatible with AppScreen's ScrollView.
