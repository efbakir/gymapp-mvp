# Unit App Audit Report
**Date:** 2026-04-05
**Build:** Unit (simulator) ‚Äî commit d617c18

## Executive Summary

The Unit codebase is in excellent shape. All feature views use `AppColor`, `AppFont`, `AppSpacing`, and `AppRadius` tokens consistently ‚Äî no raw colors, raw fonts, raw spacing, or raw corner radii were found in any feature view. Banned patterns (chevrons, bare `Divider()`, `Color.black`/`.gray`) are absent. The app is well-aligned with the product compass: no ProgressionEngine UI is surfaced in the template-based flow, and the Today screen provides a <=2-tap path to start logging. The only notable items are minor: one interstitial screen (PaywallView) uses a raw `ZStack` instead of `AppScreen`, and legacy Cycle views (unreachable from main navigation) retain `ProgressionEngine` references for data compatibility.

## Compass Alignment Score
**9/10** ‚Äî The app faithfully reflects the current compass decisions. Templates are the primary program unit, ghost values drive logging, the Gym Test is respected with large CTAs, and no cycle/engine UI is surfaced in the main flow. Minor deduction: legacy Cycle views remain in the codebase (unreachable but present).

### Minor (polish, edge cases)

1. **PaywallView missing AppScreen wrapper**
   - **Where:** `Unit/Features/Subscription/PaywallView.swift:16`
   - **What:** Uses raw `ZStack { AppColor.background ... ScrollView { ... } }` instead of `AppScreen`
   - **Rule:** `docs/atomic-design-system.md` ‚Äî "New full-screen flows should compose inside AppScreen"
   - **Mitigation:** PaywallView is a one-time interstitial shown between onboarding and main app; the raw layout replicates AppScreen behavior. Low impact.

2. **Legacy Cycle views retain ProgressionEngine references**
   - **Where:** `Unit/Features/Cycles/CyclesView.swift:449`, `WeekDetailView.swift:70`, `CycleSettingsView.swift:134`
   - **What:** Direct `ProgressionEngine.target()` / `ProgressionEngine.computeTargets()` calls in view code
   - **Rule:** `AGENTS.md` ‚Äî ProgressionEngine deferred to post-v1
   - **Mitigation:** These views are explicitly documented as "Legacy cycle views (unreachable from main navigation; retained for data compatibility)." No user can reach them.

3. **Legacy CyclesView uses raw `Color.clear`**
   - **Where:** `Unit/Features/Cycles/CyclesView.swift:403`
   - **What:** `default: return Color.clear` in `borderColor(for:)` function
   - **Rule:** `docs/atomic-design-system.md` ‚Äî prefer `AppColor` tokens over raw colors
   - **Mitigation:** `Color.clear` (transparency) is used throughout the design system itself (`DesignSystem.swift` has 6+ instances). This is standard SwiftUI idiom for "no color." Legacy view, unreachable.

4. **Simulator shows onboarding on every launch (fresh install)**
   - **Where:** App runtime ‚Äî `ContentView.swift:37` `needsOnboarding` gate
   - **What:** Cannot navigate past onboarding splash without touch interaction (simctl lacks tap support)
   - **Mitigation:** This is expected behavior for a fresh install with no data. Onboarding splash screen (`OnboardingSplashView`) is properly designed with correct tokens.
