# Unit App Audit Report
**Date:** 2026-04-02
**Build:** Unit (simulator, iPhone 17 Pro) — commit bf3630d

## Executive Summary
The app is well-aligned with the product compass. No cycle/week UI, no ProgressionEngine exposure, no social features. The design system is broadly adopted — all colors use `AppColor`, all fonts use `AppFont`, spacing and radii use tokens. Three chevron-related violations were found and fixed in this cycle. The app is near App Store ready; the primary gap is that the simulator only shows the empty-data path (onboarding splash / "Build your first program") due to a fresh data state.

## Compass Alignment Score
9/10 — All compass decisions respected. Minor deduction: `AppIcon.chevronRight` still exists in the design system enum (retained for potential future use) but is no longer referenced from any view.

## Findings

### Critical (blocks App Store submission or contradicts compass)
None.

### Major (significant UX or consistency issue)
None.

### Minor (polish, edge cases)

1. **Banned chevron icon — CalendarTabView.swift:187**
   - **What:** Month-forward navigation used `AppIcon.chevronRight` (`chevron.right`)
   - **Rule:** `goals.md` — "No chevrons anywhere in the app"; `atomic-design-system.md` — banned pattern
   - **Fix applied:** Replaced with `AppIcon.forward` (`arrow.right`)

2. **Banned chevron icon — HistoryView.swift:549**
   - **What:** Month-forward navigation used `AppIcon.chevronRight`
   - **Rule:** Same as above
   - **Fix applied:** Replaced with `AppIcon.forward`

3. **Raw system icon + banned chevrons — TemplatesView.swift:359,370**
   - **What:** Reorder buttons used raw `Image(systemName: "chevron.up")` and `Image(systemName: "chevron.down")`, bypassing `AppIcon` and using banned chevron glyphs
   - **Rule:** `goals.md` — no chevrons; `atomic-design-system.md` — use `AppIcon` for all icons
   - **Fix applied:** Added `AppIcon.moveUp` (`arrow.up`) and `AppIcon.moveDown` (`arrow.down`) to `DesignSystem.swift`; replaced raw image calls
