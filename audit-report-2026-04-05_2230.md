# Unit App Audit Report
**Date:** 2026-04-05
**Build:** Unit (simulator, iPhone 17 Pro) — commit `2bc8452`

## Executive Summary

The app is well-aligned with the product compass and design system. All active feature views use `AppColor`, `AppFont`, `AppSpacing`, and `AppRadius` tokens consistently. No banned patterns (chevrons, bare Dividers, raw colors/fonts) were found in any reachable screen. The only remaining items are legacy cycle views retained for data compatibility (unreachable from main navigation) and minor structural notes. The app is App Store ready from a design system and compass alignment perspective.

## Compass Alignment Score
**9/10** — All compass decisions from 2026-03-26 are respected in active UI. Templates are the primary unit, ghost values drive prefill, ProgressionEngine is gated behind cycle-linked sessions (which are not created in template flow). One point deducted for cycle-related data structures still threaded through `ReadyTodayContext` and `ActiveWorkoutViewModel` even though they resolve to no-ops in the template path.

## Findings

### Critical (blocks App Store submission or contradicts compass)

None.

### Minor (polish, edge cases)

1. **`ReadyTodayContext.cycleWeekNumber` field** — `TodayView.swift:36`: The `ReadyTodayContext` struct still carries a `cycleWeekNumber` field. While it defaults to `1` and is not surfaced in UI, it is dead weight that could confuse future contributors.
   - **Rule:** Compass 2026-03-26 — "8-week cycles demoted to optional layer"
   - **Suggested fix:** Remove `cycleWeekNumber` from `ReadyTodayContext` and the view model code that populates it.

2. **ProgressionEngine paths in `ActiveWorkoutViewModel`** — `ActiveWorkoutView.swift:1162-1197`: The view model calls `ProgressionEngine.target(...)` in `target()` and `prefillSet()`. These paths are inactive for template-based sessions (weekNumber=0, no cycle), but the code is still present.
   - **Rule:** Compass — "ProgressionEngine deferred to post-v1"
   - **Suggested fix:** No action needed for v1 — these paths are correctly gated and only activate for legacy cycle-linked sessions. Consider removing post-v1.

3. **Legacy Cycle views not wrapped in AppScreen** — `CyclesView.swift`, `CycleSettingsView.swift`, `WeekDetailView.swift`, `CreateCycleView.swift`: These views use `NavigationStack`/`Form`/`ScrollView` directly instead of `AppScreen`.
   - **Rule:** `docs/goals.md` — "Every screen uses AppScreen wrapper"
   - **Suggested fix:** No action — these screens are unreachable from main navigation (per AGENTS.md). Retained for data compatibility only.
