# Unit App Audit Report
**Date:** 2026-03-30
**Build:** Unit (iPhone 17 Pro simulator) — commit `00741f3`

## Executive Summary

The app is well-aligned with the product compass and design system. Previous audit cycles have resolved the major design system violations. The codebase is largely App Store ready — no placeholder text, no debug UI, no raw colors or fonts in feature views. The only structural items are legacy Cycles views (unreachable from main navigation) and a few views not wrapped in `AppScreen` (TemplateDetailView, PaywallView, and the Cycles family).

## Compass Alignment Score
**9/10** — Today screen provides ≤ 2-tap start. Ghost values are implemented. Rest timer auto-starts. Templates are the primary program unit. Cycles are demoted and unreachable from main tabs. ProgressionEngine references remain in ActiveWorkoutView for backward compatibility with existing cycle data but are not user-facing in the template-only flow. Only deduction: AGENTS.md still describes Unit as an "Adaptive Periodization Engine" and references old folder paths (`AtlasLog/`), contradicting the compass pivot.

## Findings

### Critical (blocks App Store submission or contradicts compass)

None.

### Major (significant UX or consistency issue)

1. **AGENTS.md is stale** — Still describes Unit as "Adaptive Periodization Engine" and references `AtlasLog/` folder structure (actual: `Unit/`). This misleads agents and contributors.
   - **Rule:** `docs/product-compass.md` decision log 2026-03-26 — primary promise is zero-friction logging, not adaptive coach.
   - **Fix:** Update AGENTS.md to reflect compass pivot and current folder structure. *(Not fixing in this audit cycle — doc-only change, not a code violation.)*

### Minor (polish, edge cases)

1. **TemplateDetailView not wrapped in AppScreen** — Uses `List` directly with `NavigationStack` instead of `AppScreen`.
   - **File:** `Unit/Features/Templates/TemplateDetailView.swift`
   - **Rule:** `docs/atomic-design-system.md` — "New full-screen flows should compose inside AppScreen"
   - **Note:** `List`-based views have structural constraints that make `AppScreen` wrapping non-trivial. Legacy migration path; flagged for awareness.

2. **PaywallView not wrapped in AppScreen** — Uses `ZStack` with custom layout.
   - **File:** `Unit/Features/Subscription/PaywallView.swift`
   - **Rule:** Same as above.
   - **Note:** Custom full-bleed layout may justify the exception.

3. **Legacy Cycles views unreachable but still in project** — `CyclesView`, `WeekDetailView`, `CreateCycleView`, `CycleSettingsView` are not navigable from the main tab bar but remain in the codebase.
   - **Files:** `Unit/Features/Cycles/*.swift`
   - **Rule:** Compass 2026-03-26 — cycles demoted to optional layer.
   - **Note:** Dead code. Consider removing or gating behind a feature flag in a future cleanup.

4. **ProgressionEngine references in ActiveWorkoutView** — Used for cycle-based target computation when a session has a linked cycle/rule.
   - **File:** `Unit/Features/Today/ActiveWorkoutView.swift:697,743,857,1154`
   - **Rule:** Compass — ProgressionEngine deferred to post-v1.
   - **Note:** Backward-compatible fallback, not user-facing for template-only sessions. Acceptable for data compatibility.
